───────────────────────────────────────────────────────────────
HOW IT WORKS:
───────────────────────────────────────────────────────────────

1. USER SENDS MESSAGE:
   → Message saved to database
   → is_delivered = True if recipient is online
   → is_delivered = False if recipient is offline
   → Shows clock icon initially

2. RECIPIENT RECEIVES MESSAGE:
   → Frontend emits 'message_delivered' event
   → Backend marks is_delivered = True
   → Sender sees one grey tick (✓)

3. RECIPIENT VIEWS MESSAGE:
   → Frontend emits 'message_read' event
   → Backend marks is_read = True
   → Sender sees two blue ticks (✓✓)

4. REAL-TIME UPDATES:
   → Status changes broadcast to sender via WebSocket
   → Tick indicators update instantly without page refresh
